## 1. THE TOP & THE FLOW

### THE TOP — the repeated YES
The single apex everything traces to: **the client's repeated YES** — he approves the content, it grows his business, and his trust survives every touch. Measured ONLY as `client_approved` events repeating in an append-only ledger — never an AI score, never engagement-as-proxy. *(The AI-judge scar: +0.08 vs random; no node may manufacture, predict, or proxy the YES.)*
*Source: THE_CLIENT_PYRAMID.md §1 + §4 #3.*

### THE PYRAMID LAW — build top-down, run bottom-up
His words: *"the top is what we need to achieve and we dig from the top, build from the top — full big tree on everything... we LEARN and BUILD top-down — so when we repeat the process with a real client it works bottom-up."* We lack client data, so build top-down to learn; a real client's data later fills it bottom-up. **Every thread names its TOP and its layer; no top = no work.**
*Source: CLAUDE.md Rule #5.2 / dictionary "the pyramid".*

### THE CLIENT FLOW (the 5 stages — EXTRACT → DIAGNOSE → GAPS → QUESTIONS → STRATEGY)
The agreed per-client sequence. *(Rule #11: never build ahead — diagnose, SHOW Mohamed, WAIT for the nod. Volume is not progress, his repeated YES is.)*

1. **DIAGNOSE → state classification.** Classify the brand from raw counts/dates into NEWBORN (<30 posts), DORMANT (last post >90d), ACTIVE-MESSY, or ACTIVE-STRONG. Counts/dates gate alone, but the messy/strong split is stats-PROPOSED — a **human must confirm** before any play routes on it (the +0.08 scar applies to classifiers too). State routes the entire intake.
2. **EXTRACT → one-shot intake + sector spine.** Run one clean intake (`client_intake.py`, 5 surfaces: profile / posts / comments / media / website, raw-first immutable) so everything taken from links is one less question. The sector inheritance spine silently fills ~70/80 cultural fields as *inferred* so the first proposal reads native. Coverage <90% = investigate; ZERO POSTS = error. *(Proven live: jurisha 8/8 posts, 100% coverage, 0 warnings.)*
3. **GAPS → fill the tree, name every hole.** Compose all 10 profile organs top-down from raw + voice_stats + sector defaults (`build_brand_profile.py`); every unfilled field becomes a **named gap card** with its own diagnose metric (the albaik hole becomes a field-count, not a vibe). Identity fields may **never** be inferred. *(albaik: 164 obs but red_lines absent + identity 0% = NOT production-ready — the richest profile lies loudest.)*
4. **QUESTIONS → the gap report is the ONLY questions.** The `gap_report` (5 always-ask: goals / red lines / who-speaks / USP / offer ratio + extraction-derived extras, hard cap ~7) is the ONLY thing that may be asked. Any question answerable from the 5 scraped surfaces is a **logged defect** — propose, don't interrogate. A re-ask = trust-budget violation. *(jurisha's channels are in her bio — never ask them.)*
5. **STRATEGY → voice path router + angle forge + 60-sec gate.** Route voice by state (ACTIVE-STRONG → extract + 10-pair test; NEWBORN → 3-persona birth ritual, client picks; ACTIVE-MESSY → best-20% persona + mini birth; DORMANT → extract but demote to "still true?"). Forge gated angles through deterministic Truth/Safety/offer-energy gates, then deliver ≤20 candidates as one-tap verdicts. **AI may KILL, AI may never PASS.** Statistics on 8 posts is confident fiction; a birth ritual on 164 obs is an insult — routing protects the one shot both ways.

*Sources: THE_CLIENT_PYRAMID.md §2–§5; CLAUDE.md Rule #11; transcripts d86f5bda / 02e20eb7 (Mohamed: "deginous the brand and extract evey thing first and then ask qustions").*

### THE 7 PYRAMID LAWS OF THE YES (Layer 1, v1.2 — all permanent)
What the client's eyes check, in order. Every law traces up to THE TOP.

1. **HOMEWORK LAW — "you arrived knowing me."** First contact opens with a brand mirror (products, channels, best post WITH numbers, the silence date) demonstrating ≥5 cross-verifiable facts; the proposal *demonstrates* knowledge, never requests it. *0 verifiable facts = not ready to contact.* (A pro grants one shot only to someone who already read the account.)
2. **VOICE LAW — "it sounds like us, exactly."** A filled, client-CHOSEN `l2_voice` block (voice triangle, 3 love / 3 hate lines, dialect, anti_vocabulary) confirmed by client; test = 10-pair "sounds like you?" with ≥8/10 passing. Inferred **never** silently overwrites confirmed. (One MSA caption on a Najdi family brand spends the whole shot.)
3. **TRUTH LAW — "everything stated is real."** Every noun a caption asserts (product/price/channel/branch/hashtag) must resolve to a `truth_pack.json` entry ID by deterministic lookup; an unresolved noun is a **HARD BLOCK** on render, never inferred. (Founder kill: `#ماك_مع_ججك` — "ججك what is this." One invented fact = trust death in one touch.)
4. **SAFETY LAW — "it can never embarrass me."** Two fences: client red lines verbatim + the ~10 brand-must-override cultural fields pinned at STRICTEST default until the client relaxes. Count of fields both unconfirmed AND relaxed = **0, always**; relaxation only via explicit logged client message. (Asymmetric stakes: conservative miss = one bland post; permissive miss = screenshotable public embarrassment, unrecoverable.)
5. **IDEA LAW — "there's a nameable angle."** The angle must be nameable in ONE specific sentence before rendering, built from real product × real moment × real customer words; must survive the founder_taste kills (`very_normal`, `generic_celebration_template`, `describing_the_photo`); landing judged by human eyes only. ("If any brand could post it, it's dead.")
6. **GROWTH LAW — "it sells, within my capacity."** Content matched to the client's declared goal ratio (sales vs brand), offers written clean (what / how much / until when / where), routed to real channels, capped by his capacity ceiling. Mix deviating >20% from ratio flags before reaching him; missing capacity number **hard-blocks** push content. (Growth that breaks the kitchen grows complaints.)
7. **MEMORY LAW — "I never repeat myself."** Every answer/approval/rejection/edit written immediately; the ledger is grepped before ANY outbound question; questions whose answers already existed on disk = logged trust-budget violations, **target 0 forever**. (Trust dies on the second ask — this is CURSOR.md's mid-session rule, client edition.)

*Source: THE_CLIENT_PYRAMID.md §2 Layer 1 Laws 1–7.*
