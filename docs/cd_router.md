# CD ROUTER — THE TWO-STAGE DOCTRINE (B052, June 12)
*Doc-truth law: written from the code, not from memory. Sources cited per section.*

## The doctrine in one line
**Stage 1 routes (WHO thinks), Stage 2 injects the method (HOW they think).**
A brain is never a label on the output — it's the methodology text steering the pen.

## TWO ROUTERS EXIST (by design, not drift)
### 1. The affinity router — 41-brand API path
`scripts/lib/cd_brains.py::route(sector, occasion)` → `(primary, secondary, scores)`
- Scores every brain: sector_affinity + occasion_affinity boosts − sector-forbidden set.
- cd_03 (Authenticity Detective) is EXCLUDED for captions (video/script technique) —
  the caller swaps to secondary or paradox (`api/server.py` V3 path).
- Default when brains can't load: cd_01 firaasa. Even-weight 0.2 baseline = honest
  "no brand fingerprint yet."

### 2. The deterministic router — client pipeline path
`scripts/render_client_slot.py::route_brain(slot, alt)` — occasion → brain, no scores:
- National/founding days → **heritage** (root material; falls back to firaasa when the
  client has no heritage roots — the guard lives in make_angle).
- Ramadan/Eids/Mother's Day/Hajj → **firaasa ↔ authenticity** (alt flips the pair —
  the date-hash variety door).
- competitor_reference slots → **paradox**.
- Everything else → **metaphor ↔ paradox** (alt-flipped).
Born Phase B June 11: Floward proved full methodologies beat salvaged one-liners;
determinism chosen client-side so reruns are reproducible (same slot = same brain
unless alt changes — required for pick-sets and recovery's door-change law).

## Stage 2 — the method injection
`render_client_slot.py::brain_method(brain)` reads the FULL methodology markdown
(`~/agents` CD files via BRAIN_FILES map: firaasa/metaphor/authenticity/heritage/paradox)
and injects it into the angle prompt. The angle is born INSIDE a methodology, never
labeled after the fact.

## The door-change law (B111/B112)
A rejection NEVER retries through the same door: `process_rejection.py` picks the first
brain ≠ the rejected card's brain. The retry changes the THINKING, not the wording.

## Why two routers stay two
- The API serves 41 benchmark brands with affinity data but no client organs → scored routing.
- The client pipeline serves profiled clients where reproducibility + pick-set variety
  rule → deterministic routing.
- Unifying them would force the weaker constraint on both. Revisit only when client
  fingerprints carry per-brain affinity (post-pilot).
